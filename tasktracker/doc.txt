"Task Tracker" — это веб-приложение для управления задачами. Пользователи могут создавать задачи, устанавливать для них приоритеты и отслеживать статус выполнения. Приложение также поддерживает базовые роли и разделение доступа, чтобы разные пользователи имели разные права.

Проект разработан с использованием Java и Spring Boot и демонстрирует навыки работы с REST API, базами данных и базовой системой безопасности.

Основные технологии
Java: Версия 17
Spring Boot: Версия 3.3.5
PostgreSQL: Реляционная база данных для хранения информации о пользователях и задачах
Maven: Для управления зависимостями и сборки проекта
Структура проекта
model: Классы для базовых сущностей, таких как User и Task, которые представляют данные, сохраняемые в базе данных.
repository: Интерфейсы для взаимодействия с базой данных, предоставляющие базовые CRUD-операции.
service: Сервисы для бизнес-логики, обеспечивающие основные операции с задачами и пользователями.
controller: REST-контроллеры для работы с запросами на задачи и пользователей.
security: Конфигурации для настройки безопасности и аутентификации.
config: Основные настройки приложения и безопасности.
Основные функции проекта
Работа с задачами (CRUD):

Создание, просмотр, редактирование и удаление задач.
Настройка приоритета задачи (низкий, средний, высокий) и статуса (В процессе, Выполнено, Отложено).
Фильтрация и сортировка:

Возможность фильтровать задачи по статусу и сортировать их по приоритету или дате выполнения.
Роли и доступ:

Приложение поддерживает базовые роли, такие как "Пользователь" и "Администратор".
Пользователи могут управлять только своими задачами, а администратор имеет доступ ко всем задачам.
Ограничения и нереализованные функции
Регистрация и сброс пароля: На данный момент регистрация пользователя и функционал сброса пароля не реализованы. Эти функции должны быть добавлены в будущем.
Функция подтверждения email: Проект не включает подтверждение регистрации через email.
Конфигурация проекта
Настройка подключения к базе данных:

Проект подключается к базе данных PostgreSQL.
В application.properties укажите:
properties
Копировать код
spring.datasource.url=jdbc:postgresql://localhost:5432/task_tracker
spring.datasource.username=your_database_username
spring.datasource.password=your_database_password
Запуск проекта:

После указания настроек базы данных, проект можно запустить через IDE или с помощью Maven.
Приложение будет доступно по адресу http://localhost:8080.
Основные API-эндпоинты
Получение всех задач пользователя: GET /api/tasks
Создание новой задачи: POST /api/tasks
Редактирование задачи: PUT /api/tasks/{id}
Удаление задачи: DELETE /api/tasks/{id}
Примечание: Регистрация пользователя и управление пользователями через API пока не реализованы.

Рекомендации по дальнейшей реализации
Добавить регистрацию пользователя: Создать эндпоинт для регистрации, который будет принимать имя пользователя и пароль, шифровать пароль, и сохранять нового пользователя в базе данных.
Реализовать сброс пароля: Создать механизм для сброса пароля через email или секретный вопрос.
Добавить валидацию данных: Использовать аннотации валидации (@NotNull, @Size, и т.д.) для проверки корректности данных, которые вводит пользователь.
